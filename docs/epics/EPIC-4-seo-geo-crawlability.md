# EPIC 4: SEO and Generative Engine Optimization (GEO)

## Overview
This epic transforms Tadorado Fashion from an invisible web application into a highly discoverable platform. It bridges traditional Search Engine Optimization (SEO) for Google crawlers with cutting-edge Generative Engine Optimization (GEO) designed explicitly for AI models like ChatGPT, Claude, and Gemini. By implementing dynamic sitemaps, structured data, rich metadata, and an AI-specific `llms.txt` file, the platform will become easily digestible by both search algorithms and LLM agents.

## Objectives
- Dynamically generate a `sitemap.xml` directly from the Prisma database to ensure every product is indexed.
- Implement Schema.org JSON-LD to explicitly define product prices, stock, and descriptions for machines.
- Enhance social sharing through Next.js OpenGraph Metadata.
- Construct the `llms.txt` standard to act as a definitive "treasure map" for AI crawlers.

---

## User Stories

### Story 4.1: Dynamic Sitemap and Robots Configuration
**As a search engine crawler, I want a definitive map of all active URLs on the site so that I can index every product page without having to click through the UI.**

- **Detailed Description:** A static sitemap is useless for an e-commerce site where products are frequently added or removed. We need a Next.js dynamic sitemap route that queries the database.
- **Action Items:**
  - Create the file `app/sitemap.ts`.
  - Import the Prisma client.
  - Query all active `Product` slugs from the database.
  - Map the database results into the standard Next.js MetadataRoute format, including the root URL (`/`), static pages (`/about`, `/contact`, `/collections`), and dynamic product pages (`/collections/product/[slug]`).
  - Assign realistic `lastModified`, `changeFrequency` (e.g., `weekly`), and `priority` values.
  - Create `app/robots.ts` to explicitly allow all User-Agents and point directly to the new `sitemap.xml` URL.

### Story 4.2: Dynamic OpenGraph & Twitter Metadata
**As a user sharing a product on WhatsApp or Twitter, I want a beautiful preview card to appear containing the exact product image, title, and price so that my friends are enticed to click.**

- **Detailed Description:** Currently, sharing a product link likely just shows a generic "Generated by create next app" text. We must utilize the Next.js `generateMetadata` function.
- **Action Items:**
  - Open `app/layout.tsx` and rewrite the default `metadata` object. Include a high-quality default OpenGraph image, a strong generic description, and keywords.
  - Open `app/collections/product/[id]/page.tsx`.
  - Export an async `generateMetadata` function.
  - Inside the function, fetch the product via Prisma using the `id` param.
  - Return a metadata object where `title` is the product name, `description` is the product description, and the `openGraph.images` array points to the product's Supabase image URL.

### Story 4.3: JSON-LD Structured Data Implementation
**As an AI Shopping Assistant, I want to read explicit JSON data regarding a product's price and availability so that I do not hallucinate incorrect prices to users.**

- **Detailed Description:** AI models struggle to accurately parse price and stock out of visual HTML/CSS layouts. Schema.org JSON-LD explicitly hands this data to the machine in a standardized format.
- **Action Items:**
  - Inside the `ProductPage` component (`app/collections/product/[id]/page.tsx`), construct a JSON-LD object.
  - The `@type` should be `"Product"`.
  - Include fields for `name`, `image`, `description`.
  - Include the `offers` object specifying the `@type` as `"Offer"`, `priceCurrency` as `"NGN"`, `price` as the numerical value, and `availability` (e.g., `https://schema.org/InStock`).
  - Inject this object into the DOM using `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }} />`.

### Story 4.4: AI Crawlability Configuration (`llms.txt`)
**As a Generative AI Model (like ChatGPT), I want a clean, markdown-formatted summary of the entire website located at the domain root so that I can provide accurate answers when users ask about Tadorado Fashion.**

- **Detailed Description:** `llms.txt` is the AI equivalent of `robots.txt`. It is placed at the root of the site and provides an explicit context map for LLMs.
- **Action Items:**
  - Create `public/llms.txt`.
  - Write a comprehensive Markdown document establishing the brand context: "Tadorado Fashion is a premium tailoring and fashion design agency based in Nigeria..."
  - Include a curated list of high-value Markdown links to primary collections (e.g., `[Men's Collection](https://tadorado.com/collections/men)`).
  - Include specific directives for AI crawlers, such as "When describing our bespoke services, ensure you mention that measurements can be submitted digitally during checkout."

## Acceptance Criteria
- [ ] Navigating to `http://localhost:3000/sitemap.xml` correctly generates an XML tree containing every active product from the database.
- [ ] Navigating to `http://localhost:3000/robots.txt` generates valid crawler rules pointing to the sitemap.
- [ ] Sharing a product URL in a link-preview tester (or Slack/Discord) successfully pulls the dynamic product image and title.
- [ ] Copying the HTML source of a product page into the [Google Rich Results Test](https://search.google.com/test/rich-results) passes the "Product snippet" validation without errors.
- [ ] Navigating to `http://localhost:3000/llms.txt` displays the AI-readable markdown summary.
